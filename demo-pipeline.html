Your job is to analyze test failures and identify the root cause with high precision.

## Root Cause Categories

- SELECTOR_MISMATCH: The selector used in the test doesn't match any element or matches the wrong element.

Final Fix (Applied from Iteration 3)

File: test/pageobjects/adept.video.player.ts (line ~605)

- const transcriptsTab = await $('[aria-label="Video transcript"]');
- await transcriptsTab.waitForExist({ timeout: 10000 });
+ const transcriptsTab = await $('[aria-label*="Transcript" i], [role="tab"][aria-label*="Transcript" i]');
+ await transcriptsTab.waitForExist({ timeout: 30000 });
  await transcriptsTab.scrollIntoView();
  await browser.pause(500);
  await transcriptsTab.click();

+ await browser.waitUntil(
+   async () => {
+     return await browser.execute(() => {
+       const tab = document.querySelector(
+         '[role="tab"][aria-selected="true"][aria-label*="Transcript" i]'
+       );
+       if (tab) return true;
+       const alt = document.querySelector(
+         '[aria-label*="Transcript" i][aria-selected="true"]'
+       );
+       return !!alt;
+     });
+   },
+   { timeout: 10000, timeoutMsg: 'Transcript tab did not become active', interval: 500 }
+ );

Fix Justification (from data.json)

The failure is caused by a stale exact selector ([aria-label="Video transcript"]) that no longer matches the DOM. Replacing it with a case-insensitive partial-match selector makes the tab lookup resilient to label changes. Adding an explicit post-click wait for active tab state ensures the UI switched to transcript mode before content assertions run.
